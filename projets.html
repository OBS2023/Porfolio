|AP1 : Le projet GSBfrais

Les pré-requis

Présentation de l'application GSB Frais

GSB frais est une application web permettant la gestion des frais des visiteurs médicaux des laboratoires pharmaceutiques Galaxy Swiss Bourdin (GSB).

D'un point de vu technique, l'application est codée en POO (programmation orienté objet), ce qui implique que les enrégistrement à la base de donnée sont récupérés sous la forme d'objets. Cette derniere integre la simplification des URL's (.htacces) et respecte les normes PSR-1 et PSR-4.

Model MVC

L'application utilise une architecture MVC (Model Vue Controleur).
C'est une façon d'organiser l'interface graphique d'un programme. Elle consiste à distinguer trois entités distinctes qui sont,le modèle qui contiendra les méthodes permettant d'intéragir avec la base de donnée, la vue qui conrespond à l'inerface utilisateur et le contrôleur qui reçoit tous les événements de l'utilisateur et enclenche
les actions à effectuer. Si une action nécessite un changement des données, le contrôleur demande la modification des données au modèle et ensuite avertit la vue que les données
ont changé pour que celle-ci se mette à jour. les ayant chacun un rôle précis
dans l'interface.

Model conceptuel de donnée de l'application GSB frais

Il se compose des entités : StatutFicheFrais,FraisHorsForfait, Utilisateur, Profil...
Ainsi que des associations : Déclarer, appartenir, contenir...
